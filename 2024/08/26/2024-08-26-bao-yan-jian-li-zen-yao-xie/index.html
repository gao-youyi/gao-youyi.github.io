保研简历怎么写：我把一页纸改到能投出去的过程

发表于2024-08-26|更新于2026-03-08|保研
|总字数:2.2k|阅读时长:6分钟|浏览量:

保研阶段最容易浪费时间的事情之一，就是把简历写成“材料堆砌”。

我自己本科是电子科大软件工程，最后是保送本校读计算机技术。回头看，我那份简历并不属于那种天花板级别的神仙简历，成绩也不是专业第一，科研也不是一开始就特别强，更多是靠信息整理 + 内容取舍 + 表达方式，把一页纸尽量打磨到了“老师愿意继续看”的程度。

这篇就不讲虚的，只讲我觉得真正有用的东西。

先说结论：保研简历不是自传，是筛选器

老师看你简历，很多时候没有你想象中那么细。

尤其在夏令营、预推免阶段，老师一天看几十份甚至上百份材料，第一轮看的不是“你这个人多么有故事”，而是：

  1. 你的基础盘怎么样；
  2. 你有没有明显亮点；
  3. 你和他的方向有没有一点匹配；
  4. 这人后续面试值不值得花时间。

所以，简历最重要的目标不是“全面”，而是“快速建立判断”。

如果你一页纸做不到这一点，那基本就白写了。

一页纸够不够？

大多数同学，够。

除非你真的是论文、专利、项目、竞赛都很多，而且每一项都很硬，不然我还是建议一页。因为保研简历和找工作简历一样，核心就是两个字：克制。

我比较推荐的一页结构如下：

1. 基本信息：别花里胡哨

放这些就够了：

  • 姓名
  • 手机号
  • 邮箱
  • 本科学校、专业
  • 英语情况（六级）
  • 个人主页 / GitHub（如果真的有内容）

不要在最上面堆一堆没有意义的信息，比如身份证号、政治面貌大段描述、家庭住址之类。政治面貌如果你想写，可以很简短地放一行，我自己简历里也写过中共党员，但它绝对不是主角。

邮箱别用奇怪昵称，尽量学校邮箱或常规邮箱。文件名也别叫“最终版2-真的最终版.pdf”，建议直接：

2. 教育背景：把“保研相关信息”写明白

这一块很多人写得太像流水账。

建议重点写：

  • 学校、专业、时间
  • 排名或前百分比
  • GPA（如果好看）
  • 英语成绩
  • 主要荣誉

如果你的排名不是顶尖，但也不差，比如专业前 10%～15% 左右，那就老老实实写“专业前 xx%”。不要模糊写“成绩优良”“排名靠前”，这种话等于没写。

我自己的体会是：排名、英语、竞赛级别，这几个东西最好让老师扫一眼就能看见。

3. 科研经历：没有论文，也要写得像科研

很多本科生会卡在这里：我没有正式论文，怎么写科研？

我的看法是，别把科研狭义理解成“发表”。

如果你做过：

  • 老师带的项目
  • 课程综合设计里偏研究型的部分
  • 自己复现过论文并做过改进
  • 有相对完整的实验过程和结果分析

这些都可以写，而且完全值得写。

但写法很关键，不要写成：

参与某某课题，主要负责代码。

这句话几乎没有任何信息量。

更好的写法应该是：

  1. 先说问题背景；
  2. 再说你具体负责什么；
  3. 最后说结果或产出。

比如我现在回头看自己更成熟一点的写法，会更偏向这种：

  • 针对某类时空预测任务中的噪声与缺失问题，参与模型设计与实验验证；
  • 负责数据处理、基线复现、消融实验和结果分析；
  • 在 PeMS 等数据集上完成多组对比实验，支撑后续论文撰写。

哪怕你当时只是本科生，这样写也比“参与科研项目”强很多。

4. 项目经历：不要写“做了一个系统”，要写“你解决了什么”

如果你和我一样，本科阶段竞赛和项目会比纯科研更多，那项目经历一定要认真写。

我自己的经验是，项目部分最容易拉开差距。因为很多同学项目确实做了，但不会写。

一个项目至少回答三个问题

  • 这个项目是干什么的？
  • 你负责的是哪一部分？
  • 最终效果怎么样？

比如不是写：

参与开发某某系统，主要负责后端。

而是写：

  • 面向 xx 场景设计并实现 xx 功能模块；
  • 基于 xx 技术栈完成前后端联调 / 数据库设计 / 模型部署；
  • 支撑项目在某比赛中获奖，或支持 xx 用户流程完整跑通。

我个人会优先保留两类项目：

第一类：有比赛结果的项目

比如国家级、省级竞赛项目。这种项目天然自带“第三方认证”。

第二类：技术上能讲出细节的项目

比如你真的做过全栈、算法、部署、模型优化，那面试中很容易被追问，也更容易让老师觉得你不是只会 PPT。

5. 奖项荣誉：只放有用的

奖项这块不要全塞。

我建议按优先级排序：

  1. 国家级竞赛奖
  2. 省部级 / 校级中含金量高的奖
  3. 与专业能力相关的荣誉
  4. 其他综合荣誉

如果你奖很多，也不要把“参加过”“入围过”写得太满。老师对“获奖”和“参与”的判断完全不是一个量级。

像“电子科大优秀毕业生”“优秀个人”“优秀共青团员”这类可以放，但不要让它们压过更硬的竞赛和项目。

6. 技能栏：别写成关键词大乱炖

很多人技能栏喜欢写一排：

Python / C++ / Java / Linux / Docker / PyTorch / TensorFlow / MySQL / 前端 / 后端 / 大模型 / 数据分析 / 算法设计……

看起来很强，实际上像在许愿。

我建议技能栏只写你真的能在面试中接得住的：

  • 编程语言：Python、C++、Java（择主写）
  • 开发工具：Git、Linux、Docker
  • 深度学习：PyTorch
  • 其他：LaTeX、Prisma、PostgreSQL、Nginx 之类

数量不在多，在于可信。

7. 一页纸里最该强调的，是“量化”

这一点太重要了。

老师天然对量化信息更敏感，因为好判断。

比如：

  • 专业前 15%
  • CET-6
  • 国家二等奖 / 国家三等奖
  • 一作在投 / 二作已发表
  • 模型参数量 0.24M
  • 项目从 0 到 1 搭建完成 MVP

哪怕不是所有经历都能量化，也尽量在关键处加上结果和规模。

8. 简历里最忌讳的几个问题

（1）废话太多

“本人具有较强的团队协作能力、沟通能力、执行能力……”

这种句子建议删掉。没有证据的优点，不如不写。

（2）只写做了什么，不写做成什么

做过程不等于有结果。简历是结果导向的。

（3）顺序混乱

老师最烦找信息。你的排名、竞赛、项目、科研，最好都摆在该摆的位置。

（4）排版太挤

不是内容越多越厉害。行距、留白、字号都要让人愿意看。

9. 我最后是怎么取舍内容的

我自己后来做简历时，基本遵循一个原则：

不能帮助老师在 30 秒内判断我值不值得面的内容，就尽量删。

所以我更愿意保留：

  • 学校专业和基本盘
  • 代表性的国家级竞赛
  • 能讲出技术细节的项目
  • 有明确方向关联的科研经历

而不是把所有经历都堆上去。

10. 如果你现在就要开始改简历，我建议按这个顺序来

第一步：列清单

把你所有能写的东西先列出来，不要急着排版。

第二步：筛选

每一条都问自己一句：

这条如果删了，会不会影响老师判断我的能力？

不会，就删。

第三步：改成结果导向的句子

每一条尽量写成“背景 + 动作 + 结果”。

第四步：控制到一页

删到心疼，往往才是刚好。

最后说一句

保研简历没有什么一招制胜的神技。

绝大多数时候，它拼的不是修辞，而是你有没有把自己真正有价值的经历，按老师能快速理解的方式呈现出来。

说白了，简历不是让你感动自己，而是让别人愿意给你下一轮机会。

如果你现在正准备夏令营、预推免，别一直纠结模板和配色，先把内容打磨明白。那才是最值钱的。

LeetCode Hot100 刷题经验：不是刷完 100 道就够了，而是要形成模板
我最近又把 LeetCode Hot100 过了一遍。 不是因为它有多神，而是因为到研究生阶段你会发现：算法题这东西，一段时间不碰，手感掉得非常快。尤其是平时做科研、写工程代码比较多的时候，脑子会更习惯“大系统逻辑”，反而不够敏感于题目里的局部结构。 所以这篇不写“我刷了多少题”，只写我觉得真正有用的经验。 1.
